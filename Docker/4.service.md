# 使容器开机自启动

推荐做法是使用 systemd 管理

1. 编写一个 .service 文件，内容大致如下：

    ```ini
    [Unit]
    Description=CRMEB Redis Container
    After=network.target

    [Service]
    Type=simple
    Restart=always
    RestartSec=5

    # 启动命令在这里
    ExecStart=/usr/bin/podman run \
    --name crmeb-redis \
    --rm \
    --pod crmeb-pod \
    -v /www/wwwroot/kafei-redis-data:/data:z \
    redis:7-alpine \
    redis-server --appendonly yes

    ExecStop=/usr/bin/podman stop -t 30 crmeb-redis
    ExecStopPost=/usr/bin/podman rm crmeb-redis

    [Install]
    WantedBy=multi-user.target
    ```

2. 启用该服务
    
    注意：如果是用户自己的服务，应使用 systemctl --user

    - sudo systemctl daemon-reload 
        - 重新加载配置
    - sudo systemctl enable --now container-my-nginx.service
        - 设置开机自启动，并立即启动（--now）
    
3. 查看状态

    - 查看容器状态 `podman ps`
    - 查看 Systemd 服务状态 `sudo systemctl status container-my-nginx.service`
    - 查看是否已启用开机自启动 `sudo systemctl is-enabled container-my-nginx.service`
    - 停止容器 `sudo systemctl stop container-my-nginx.service`
    - 禁用开机自启动 `sudo systemctl disable container-my-nginx.service`

# 注意

对 podman 来说，systemctl 启动服务是否 sudo，决定容器是 rootful 还是 rootless

- systemctl --user：Rootless
- sudo systemctl：Rootful
