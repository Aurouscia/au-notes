# ssh-key-gen

免密登录 ssh 的最简单方案，且不会影响原登录方式

## 1.生成密钥对

```sh
ssh-keygen -t ed25519 -C "your_email@example.com"
# 按回车，使用默认路径 & 不设密码
```

- `-t` 表示加密算法，推荐使用 ed25519
- `-C` 表示注释，可标明密钥使用者身份

按第一下回车时，可指定密钥文件名（默认为`id_ed25519`，如果生成多个密钥会冲突）

公钥文件名为私钥文件名+.pub，二者应保持对应

## 2.公钥发给服务器

```sh
ssh-copy-id user@remote_host
```

如果上述工具没有，可以改为手动操作：

- 确保服务器有 .ssh 目录
    - `mkdir -p ~/.ssh && chmod 700 ~/.ssh`
- 移动公钥
    - `scp .\.ssh\id_xxx.pub user@remote_host:/tmp/id_xxx.pub`
    - windows 的 .ssh 目录在 USERPROFILE 下
- 追加到 authorized_keys 文件
    - `cat /tmp/id_xxx.pub >> ~/.ssh/authorized_keys`

## 3.修改客户端的 ~/.ssh/config 文件

默认情况下会尝试使用 id_ed25519, id_rsa 等默认私钥文件名

如果文件名是自定义的，则需要在 ssh 配置文件显式指定

```
Host myserver # 服务器标识名称
  HostName 12.34.56.78 # 服务器IP
  User root
  Port 22
  IdentityFile ~/.ssh/xxx # 私钥文件路径
  IdentitiesOnly yes # 只使用上述给定私钥文件，不尝试其他的
```

以后连接时如果用 `ssh myserver`（参数与配置中的 Host 一样），则会使用指定的私钥文件
