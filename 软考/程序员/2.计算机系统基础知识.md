# CPU
- 运算器 ALU
    - 图上一般画成梯形
    - 执行算术运算（如加减乘除）和逻辑运算（如与、或、非）
- 控制器 CU
    - 负责从内存中取指令、译码，并控制其他部件协同工作
- 寄存器组 Registers
    - 高速存储单元，用于临时存放指令、数据和中间结果
- 缓存 Cache
    - 分为L1，L2，L3级
    - 用于减少访问主存的延迟
- 总线接口单元

- 内部总线 on-chip bus
    - 数据总线
    - 地址总线
    - 控制总线
## 连接关系
```
CU - 总线
|
ALU - 寄存器组
|
Cache - 总线接口 - 主存
```
## 控制流程
- 取指令：CU取指令（通过PC（程序计数器）从内存取指令到指令寄存器）
- 译码：CU译码，确定操作类型和操作数位置
- 执行：ALU从寄存器或缓存取数据，执行，写回寄存器
## 多核
分治，在特定时间内完成更多任务

# 存储
## 按工作方式
- RAM 随机存储器（可读可写，易失性）
- ROM 只读存储器（非易失性）
    - ROM 固定~ （出厂时就内容固定，不可更改）
    - PROM 可编程~ （让用户编程一次，之后不可更改）
    - EROM 可擦除~ （可用紫外线照射擦除）
    - EEPROM 带电可擦除可编程~ （可用电信号擦除）
        - 也可以写作E2PROM（同一个东西）
    - FlashMemory 闪存（同上，但可按块擦除）

## 按寻址方式
- RAM 随机存储器（固态硬盘（本质上是Flash芯片））
- SAM 顺序存储器（磁带）
- DAM 直接存储器（机械硬盘）

## 层级
从上到下：容量越来越大、成本越来越低
- CPU 寄存器
- CPU 高速缓存
- 主存（内存）
- 辅存（外存）

## 主存
可寻址单元个数：2^(地址寄存器的位数)   
性能指标： 
- 容量
- 存取时间（从接到存取命令到完成的时间）
- 带宽
- 可靠性
    MTBF（平均故障间隔时间）

## 高速缓存 Cache
- 需要它的原因：
    - 寄存器与主存的速度不一致
    - 程序执行的局部性
        - 时间局部性：某些指令会反复执行
        - 空间局部性：某些数据需反复访问
- 特点
    - 容量小（介于寄存器和主存之间，一般几M）
    - 速度块（比主存快5-10倍）
    - 透明
- 组成：控制部分、存储部分
- 命中：直接取Cache
- 未命中：将Cache中的数据替换，或将数据写入Cache

## 外存
- 硬盘
    - 磁盘（机械）圆盘+读写头
    - 固态
    - 混合
- 光盘
    - CD-ROM 只读
    - WORM 用户可写一次
    - 可擦除型
    - 特点：非接触、耐用、可靠、无需防震除尘、速度较慢
- U盘
    - USB移动硬盘、闪存盘
    - 支持热插拔，取代软盘
- 云存储

# IO
- 接口：主机与IO设备的转换机构
    - 主要功能
        - 地址译码
        - 交换指令与状态
        - 支持中断和DMA
        - 缓冲和驱动
    - 分类
        - 串行：近距离高速
        - 并行：远距离低速
        - 同步/异步
- I/O系统
    - 连接方式
        - 总线型（通过协议实现分时共享）
        - 通道
        - I/O处理机
    - 编址方式
        - 与内存单元统一编址（I/O中的存储看作主存中的单元）
        - I/O接口单独编址（设置专门的IO指令访问）
    - 数据交互方式
        - 直接程序控制（下分两类）
            - 程序查询方式：cpu空转，不断询问外设是否准备好
            - 无条件传送：cpu认为外设始终可读写，直接进行操作
        - 中断方式（外设的引起中断的事件（中断源）使得CPU中止当前程序去处理事件）性能严重下降
        - 直接存储器存取（由DMA（Directe Memory Access）控制，**无需cpu干预**）
        - 通道控制方式（专用控制器，cpu只需按约定格式启动通道即可）
