# 四种覆盖准则
由弱到强：

- 语句覆盖（Statement Coverage）
    程序中的每一个statement（正方形）都被执行至少一次
- 条件覆盖（Condition Coverage）（零件换档）
    程序中的每一个condition（菱形）**条件表达式**的每个可能的**取值**（真/假）至少满足一次
- 判定/分支覆盖（Decision/Branch Coverage）（整机换档）
    程序中的每个branch（菱形的条件）的每个**分支**至少执行一次，关注的是判定表达式的结果
- 路径覆盖（Path Coverage）
    程序中的每个path至少经过一次，例如两个无关的if-else，则有2*2=4条路径  
    对于有循环的流程图，路径考虑1.不跳回 2.跳回一次 两种情况（例题里面是这样）

分支覆盖强于条件覆盖的例子：  
例如`a||b`，a真b假/a假b真，足以满足条件覆盖，但会漏掉else分支

# 分层
- 算法层：用于测试类中定义的每个**方法**，基本相当于传统软件测试中的单元测试。
- 类层：用于测试封装在同一个类中的所有**方法与属性之间的相互作用**。
    在面向对象软件中，类是基本模板，因此可认为这是面向对象测试中所特有的模块测试。
- 模板层：用于测试一组协同工作的**类之间的相互作用**。
    大体上相当于传统软件测试中的集成测试。但这个也有面向对象的特点，如对象之间通过发送消息相互作用。
- 系统层：把各个子系统组织完成的面向对象的软件系统，在组装过程中进行测试。

# 类型
- 验证测试：在一个模拟环境中使用模拟数据运行系统，主要寻找错误和遗漏
- 确认测试：在一个实际环境中使用真实数据运行系统
- 审计测试：证实系统没有错误并准备好了可以运行

# McCabe 方法（McCabe’s Cyclomatic Complexity）
用来量化一段代码的控制流（分支、循环）复杂程度
- C(G) = 判定节点数 + 1（判定节点包括if、while、for等，for的“离开循环条件”也算一个）
- C(G) = 封闭区域数 + 1（仅对平面图且 联通分量=1 成立）
