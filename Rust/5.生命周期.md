# 生命周期
`'`符号用来标识生命周期

## 字符串字面量
字面量`"ABC"`实际上是
```rs
&'static "ABC"
```
`&'static`表示其是指向静态内存的引用，生命周期和整个程序一样长

## struct内的
- 不能写：
    ```rs
    struct Holder {
        text: &str,
    }
    ```
- 应该写：：
    ```rs
    struct Holder<'a> {
        text: &'a str,
    }
    ```
    表示：`text`的生命周期和其所属的`Holder`对象一样长

## 自动解引用
为什么`&"ABC"`（也就是`&&'static "ABC"`，双层引用）能赋值给`&str`？  
这是由于：rust自动剥了一层`&`，使类型匹配（尽量不要让这种事发生，以免降低可读性）

## 需要生命周期标注的情况
- 如果返回一个`String`，发生所有权转移，那么不指定生命周期，因为数据的所有权已经转给调用方
    ```rs
    fn create_string() -> String {
        let s = String::from("hello");
        s  // ✅ 合法：转移所有权
    }
    ```
- 如果返回一个`&str`，不发生所有权转移，那么必须指定这个引用的生命周期，否则是悬垂引用
    ```rs
    // ❌ &str：生命周期不明确，编译错误
    fn bad_str() -> &str {
        let temp = String::from("temp");
        &temp  // temp 会被释放，返回悬垂引用
    }
    ```
- 返回一个`&str`时，必须使其生命周期明确
    ```rs
    // ✅ &str：生命周期明确，安全借用
    fn get_part<'a>(full: &'a str) -> &'a str {
        &full[0..3]  // 返回的 &str 与输入参数生命周期相同
    }
    ```
