# cni-podman0

注：新版 podman 在逐渐废弃 CNI 支持，老版的才这么用

podman 在 CNI（container network interface）模式下，会在宿主机创建一个网桥

默认位置是 `10.88.0.1`，使用 `10.88.0.0/16` 子网，然后把容器和 pod 分配到这个子网的 IP 下，例如 10.88.0.12

网桥的位置不会变化，即使 podman 重启或宿主机重启

## 容器访问宿主

如果宿主有 mysql，而容器内需要访问它，可以填 `DB_HOST = 10.88.0.1`，即可访问到宿主的 3306 端口的 mysql 实例
（也要设置 mysql 允许这么连接）

## 宿主允许容器

对于 mysql，使用 `user@10.88.%` 这种 host 通配符，才能从子网中的容器访问

对于 firewalld，使用 `--add-source=10.88.0.0/16`，设置到 zone 中并允许连接 3306端口
