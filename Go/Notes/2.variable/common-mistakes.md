# 错题集：变量与常量

## 一、无类型常量（Untyped Constant）

**错误题目**：`const Pi = 3.14` 中，Pi 的类型是 float64。

**错误答案**：对

**正确答案**：错，Pi 是无类型常量（untyped constant），不是 float64

**知识点**：
- 使用 `const` 声明常量时，如果不显式指定类型，得到的是**无类型常量**
- 无类型常量可以参与多种类型的运算，编译器会根据上下文自动推导
- 相比之下，`var a = 3.14` 会固定得到 `float64` 类型

---

## 二、短变量声明 `:=` 的重复使用规则

**错误题目**：以下代码是否合法？
```go
x := 10
x, y := 20, 30
```

**错误答案**：错，x 重复声明

**正确答案**：对，短变量声明中只要至少有一个变量是新声明的，就可以重复使用已存在的变量

**知识点**：
- `:=` 短变量声明允许重复使用已存在的变量，但条件是**至少有一个变量是新声明的**
- 已存在的变量会被重新赋值，而不是重复声明

---

## 三、var 与 := 的使用场景

**错误题目**：什么情况下应该使用 `var` 声明变量，什么情况下应该使用 `:=`？

**不完整答案**：
- 函数内可以使用`var`也可以使用`:=`
- 函数外只能使用`var`

**补充要点**：
- 使用 `var` 可以**显式指定类型**
- 使用 `:=` 编译器会**自动推断类型**，且**必须初始化**

## 四、值接收者方法修改无效

**错误题目**（结构体部分）：
```go
type Counter struct {
    count int
}

func (c Counter) Increment() {
    c.count++
}

func main() {
    c := Counter{count: 0}
    c.Increment()
    c.Increment()
    fmt.Println(c.Value())  // 输出？
}
```

**错误答案**：2

**正确答案**：0

**原因**：`Increment()` 使用的是**值接收者**，方法内修改的是副本，不会影响原结构体

**修复方法**：将方法改为指针接收者
```go
func (c *Counter) Increment() {
    c.count++
}
```

---

## 总结

| 错误类型 | 关键知识点 |
|---------|-----------|
| 无类型常量 | `const` 不指定类型时得到无类型常量，不是固定类型 |
| 短变量声明 | `:=` 允许重复使用变量，只要至少有一个新变量 |
| var vs := | `var` 可显式指定类型，`:=` 自动推断且必须初始化 |
| 值接收者 | 值接收者方法修改的是副本，不影响原结构体 |
