# 连接的种类
- LEFT JOIN：左连接，左表的所有记录都会出现在结果集中，右表中没有匹配的记录会显示为 NULL
- RIGHT JOIN：右连接，右表的所有记录都会出现在结果集中，左表中没有匹配的记录会显示为 NULL
- INNER JOIN：内连接，只返回两个表中匹配的记录，任意一边没有匹配的记录都会被过滤掉
- ~~FULL OUTER JOIN~~：全连接，返回左表和右表的所有记录，无论是否匹配都显示（mysql不支持）
- CROSS JOIN：交叉连接，返回左表和右表的笛卡尔积，即所有可能的组合（不能有ON字句）  
    在mysql中，cross join可以带on子句，且效果与inner join相同（最好不要用）

# 例子
```sql
SELECT pets.name, pets.species, assets.owner, assets.price
FROM `pets` RIGHT JOIN ( /* 右连接：右侧的行必须都存在 */
    SELECT * FROM assets
    WHERE price > 10
) AS assets /* 为子查询起AS别名（必须） */
ON pets.name = assets.owner 
ORDER BY assets.price;
```
