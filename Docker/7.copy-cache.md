# copy

```dockerfile
COPY . /var/www
```

COPY 命令会将本地上下文中的指定目录复制到 docker 上下文的指定目录

被 .dockerignore 排除的目录不会存在于 COPY 的目标中，强行点名里面的文件会导致 build 失败

## 利用缓存的失效尝试

```dockerfile
COPY ./some-dir /var/www/some-dir
COPY . /var/www/some-dir
```

这样看起来似乎可以在 some-dir 之外的文件变化时，避免 some-dir 那层重新推送，但其实并没有用，因为第二层里也包含着 some-dir。

类似的，将内容 COPY 进去后再删掉 some-dir，也无法让缓存无视 some-dir 的变化

要形成有效的缓存，应该确保完全分离：

- 一个个 COPY 所有条目
- 分到两个大目录中，分别 COPY
