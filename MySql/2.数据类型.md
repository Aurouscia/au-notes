# 数值
修饰符：
- `UNSIGNED` 整数类型可在后面添加，除了整数之外的其他数值类型不可以加（已废弃）
- `AUTO_INCREMENT` 整数类型可在后面添加，插入null或0时，该列会自动取“当前最大值+1”的值

## 整数
字节数 名称
- 1  `TINYINT`
- 2  `SMALLINT`
- 3  `MEDIUMINT`
- 4  `INT`
- 8  `BIGINT`
注：整数类型不应在后面加括号（已废弃）  

## 布尔
`BOOL`和`BOOLEAN`其实是`TINYINT`（占1字节）  
- 表达式`FALSE`的值为0  
- 表达式`TRUE`的值为1
- 在布尔上下文中：（需要判断真假）
    - falsy: 0, NULL
    - truthy: 所有其他值

## 定点
- `DECIMAL(m, n)`  十进制小数
- `NUMERIC`  是个alias与前者完全一致

例子：DECIMAL(5,2) 可存储 -999.99 到 999.99  
注：DECIMAL(5) 相当于 DECIMAL(5, 0)  
注：DECIMAL 相当于 DECIMAL(10, 0)  
注：m最大65

## 浮点
- 4  `FLOAT`
- 8  `DOUBLE`

注：浮点类型不应在后面加括号（意义不大）  
注：避免将浮点类型（不准确）做比较运算

## 比特
`BIT(m)`  
m可是1-64  
可通过`b'101010'`的形式表示BIT字面量  
例如`b'10000000'`表示128

# 字符串

# NULL
`WHERE a = NULL`：错误  
`WHERE a IS NULL`：正确
- 定义：NULL 表示“未知值”或“缺失值”。
- 比较：任何与 NULL 的比较（=, !=, <, > 等）结果都是 NULL，即 未知。
- 判断：使用 `IS NULL` 或 `IS NOT NULL` 来判断是否为 NULL。
- 太空船运算符：`NULL <=> NULL`为TRUE
- 运算：大多数运算（加减乘除、字符串拼接等）中，只要有一个操作数是 NULL，结果就是 NULL。
- 聚合函数：SUM()、AVG()、COUNT(col) 会 忽略 NULL；但 COUNT(*) 不会忽略。
- 索引：MySQL 的 BTree 索引 不会索引 NULL 值（除非是某些特定引擎如 InnoDB 的二级索引）。
- 唯一约束：在 UNIQUE 索引中，允许多个 NULL（因为 NULL != NULL）
