# 1NF
表中每个字段都必须是原子性的，即不可再分。
每行每列只能存储一个值，不能是列表、数组、逗号分隔的字符串等。
- 违反示例：
  - 姓名："张三,李四"
  - 爱好："游泳,篮球,足球"
  - 地址："北京,上海,广州"

# 2NF
在1NF的基础上，表中的*非主键字段*必须**完全**依赖于主键，不能存在部分依赖。
即非主键字段不能依赖于主键的一部分。
- 违反示例：
  - 产品ID：1001 # 主键
  - 厂家ID：2001 # 主键
  - 库存数量：100
  - 厂家地址："北京"        # 主键为产品ID和厂家ID，非主键字段"厂家地址"仅依赖于"厂家ID"，不完全依赖于主键

# 3NF
在2NF的基础上，表中的*非主键字段*必须**直接**依赖于主键，不能存在传递依赖。
即非主键字段不能依赖于其他非主键字段。
- 违反示例：
  - 订单ID：1001 # 主键
  - 客户ID：2001
  - 产品ID：3001
  - 订单日期："2022-01-01"
  - 产品名称："手机"        # 主键为产品ID，但非主键字段"产品名称"仅依赖于"产品ID"，而不直接依赖于主键

# BCNF
在3NF的基础上，表中的每一个非平凡函数依赖“X → Y”，**X都是表的一个超键**（即 X 能唯一标识表中的每一行）

# 4NF
在 BCNF 的基础上，消除表中存在的非平凡且非函数依赖的**多值依赖**
- 多值依赖（MVD）：表示一个属性集的值可以决定另一个属性集的多个独立值。记作 X ↠ Y。
- 违反示例：
  - 数学、张老师、教材A
  - 数学、张老师、教材B
  - 数学、李老师、教材A
  - 数学、李老师、教材B
老师和教材但都多值依赖于科目，所以不符合4NF（会造成数据冗余）
