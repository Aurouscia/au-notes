自动加载
控制器类(可增加)，模型类(可增加)，框架核心类
分为两类：
位置可以确定不动的类（当自动加载方法确定时，类文件确定了）
可以扩展的类（没有办法确定）
对于已经确定的类，用最简洁的方式加载
对于需要扩展的类，需要通过类名的规律，完成位置的判断，进而完成自动加载
function userAutoLoad($className){
    //做一个类名与类文件的映射
    $framework_class_list=array(
        'controller'=>'./framework/controller.class.php',
        'model'=>'./framework/model.class.php',
        'Factory'=>'./framework/Factory.class.php',
        'sqlObj'=>'./framework/sqlObj.class.php'
    );
    if(isset($framework_class_list[$className])){
        require $framework_class_list[$className];
    }
    //控制器类，截取后十个匹配字符串
    elseif(substr($className,-10)=='controller'){
        require './application/'.p.'/controller/'.$className.'.class.php';
    }
    elseif(substr($className,-5)=='model'){
        require './application/'.p.'/model/'.$className.'.class.php';
    }
}

跳转：
立即跳转和等一会再跳转
Header('Location:');
PHP服务器向浏览器发送一条指令（响应头信息）
header函数前不应该有任何输出东西，否则会占用输出缓冲
header函数后的东西也会照常执行完毕(哪怕跳转了)
所以跳转后面应该立即终止脚本，header()后面接上一个die;
n秒后跳转：
Header('Refresh:秒数;URL=目标');
会在当前URL上停留几秒，之后就会跳转
提示的样式可以轻松定制
封装一个跳转的方法，是所有控制器都需要的功能（应该放到基础控制器里）
