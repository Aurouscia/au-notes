# 主键
## 添加主键  
`ALTER TABLE <表名> ADD PRIMARY KEY (<列1名>, <列2名>...)`  
```sql
ALTER TABLE t ADD PRIMARY KEY (a,b,c);  
```
> 为什么需要括号包裹：因为ADD PRIMARY KEY后跟**一组**列名，即使单个也需要括起来

## 创建表时指定外键
跟在列名后面（修饰这个列的性质）  
`CREATE TABLE t (id INT PRIMARY KEY, ...)`

# 外键
## 添加外键 
`ALTER TABLE <子表名> ADD CONSTRAINT <外键名> FOREIGN KEY (<子表列名>) REFERENCES <主表名>(<主表列名>);`  
```sql
ALTER TABLE orders
    ADD CONSTRAINT fk_user
    FOREIGN KEY (user_id) REFERENCES users(id);
```
- 注：主表中被refer的列，必须有唯一性（或主键）的约束

## 创建表时指定外键
以`CONSTRAINT`关键字开头，写在列名列表中
```sql
CREATE TABLE assets (
    `id` INT AUTO_INCREMENT PRIMARY KEY,
    `owner` VARCHAR(20) NOT NULL,
    `name` VARCHAR(20) NOT NULL,
    `remark` VARCHAR(20),
    CONSTRAINT `fk_assets_owner`
        FOREIGN KEY (owner) REFERENCES pets(name)
        ON UPDATE CASCADE
        ON DELETE CASCADE
    );
```
外键的性质：`ON UPDATE`和`ON DELETE`后可跟：
- 数据必须同生共死 → `CASCADE`
- 严禁误删，先清依赖 → `RESTRICT`
- 主表记录可消失，从表留档 → `SET NULL`
