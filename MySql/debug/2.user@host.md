# user@host

在 mysql 中，用户以用户名+来源主机的方式存储，例如`root@localhost`

`'admin'@'123.4.5.6'` 和 `'admin'@'localhost'` 是两个不同的用户，即使密码相同，也必须用对应的主机连接，且可以有不同的权限配置

## 查看所有用户

```sql
SELECT user, host, plugin, authentication_string FROM mysql.user;
```
得到：
```
+------------------+-----------+-----------------------+-----------------------+
| user             | host      | plugin                | authentication_string | 
+------------------+-----------+-----------------------+-----------------------+
| root             | localhost | caching_sha2_password | ...                   |
| root             | 127.0.0.1 | caching_sha2_password | ...                   |
| debian-sys-maint | localhost | caching_sha2_password | ...                   |
| app_user         | 192.168.% | caching_sha2_password | ...                   |
| admin            | %         | caching_sha2_password | ...                   |
+------------------+-----------+-----------------------+-----------------------+
```

## 添加 host 来源

如果要允许某用户名从其他 IP 登录，需要修改 HOST 限制或添加新用户

- `%` 表示允许任何来源（不安全）
- `10.88.%` 表示允许 10.88 开头的网段

```sql
RENAME USER 'olduser'@'localhost' TO 'newuser'@'%';
CREATE USER 'user'@'新主机' IDENTIFIED BY '密码';
```

然后运行“查看所有用户”，如果 authentication_string 一样说明密码一样
