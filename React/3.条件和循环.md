# 条件语法
## a ? b : c
三目运算符：使用?和:来条件性地渲染内容。
```tsx
return (
  <li className="item">
    {isPacked ? name + ' ✅' : name}
  </li>
);
```
也可以：在三目运算符中直接写元素
```tsx
return (
    <>
        {cond ? <A /> : <B />}
    </>
)
//表示 “当 cond 为真值时, 渲染 <A />，否则 <B />
```

## &&
逻辑与运算符：使用&&来条件性地渲染内容。
如果&&号左侧为truthy，则渲染右侧的内容，否则渲染左侧的内容（所以左侧不能为数字0）。
```tsx
return (
    <li className="item">
        {name} {isPacked && '✅'}
    </li>
);
```

# 循环语法
循环：可以使用数组的map方法将其过滤并转换为元素
```tsx
const listItems = [
    {id:1, text:'发明一种新式信号灯'},
    {id:2, text:'排练电影场景'},
    {id:3, text:'改进频谱技术'}
]
const listElements = listItems.map(x=><li key={x.id}>{x.text}</li>)
return <ul>{listElements}</ul>
```

## key
key属性：必须每个元素唯一，否则会出现各种问题。
key属性：必须在循环元素上，不能在内层（即使是虚拟的）
不能把循环索引当作key，除非顺序不会改变。
### 错误写法
```tsx
function Recipe({recipe}){
    return <div key={recipe.id}>{recipe.name}</div>
}
function RecipeList(){
    return (
        <h1>菜谱</h1>
        {recipes.map(r =>
            <Recipe recipe={r}/>
        )}
    );
}
```
### 正确写法
```tsx
function Recipe({recipe}){
    return <div>{recipe.name}</div>
}
function RecipeList(){
    return (
        <h1>菜谱</h1>
        {recipes.map(r =>
            <Recipe key={r.id} recipe={r}/>
        )}
    );
}
```

## Fragment
如果需要在循环中重复渲染多个元素为一组  
可以使用Fragment元素（即<></>的全称写法）  
避免出现多余的div元素，提高渲染效率。  
不能使用<></>写法，因为它不能指定key属性。
```tsx
const listItems = people.map(person =>
  <Fragment key={person.id}>
    <h1>{person.name}</h1>
    <p>{person.bio}</p>
  </Fragment>
);
```
