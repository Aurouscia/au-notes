# let
## 可变/不可变
- `let`不可变，类似js的const
- 需要可变的，必须显式声明`let mut`
```rs
let x = 1;
let mut y = 1;
```

## 类型
- 使用let+立即赋值，自动推断类型
- 使用类似ts的方式显式标注类型
```rs
let x = 1; // 数字默认为 i32
let y: u64 = 2;
```
### 基本类型
- `u8`, `u16`, `u32`, `u64`, `u128`, `usize`：无符号整数，size表示内存指针宽度（32或64）
- `i8`, `i16`, `i32`, `i64`, `i128`, `usize`：有符号整数
- `f32`, `f64`：单、双精度浮点数（默认为f64）
- `bool`：布尔
- `char`：4字节unicode字符
- `()`：空元组/单元类型（相当于函数的void）
- `String`：字符串（堆分配，可变，拥有所有权）
- `&str`：字符串切片（借用的，不可变）
- `[T; N]`：数组（固定长度）
- `&[T]`：数组切片

## `Option<T>`
表示一个可选的某种类型
```rs
let maybe_number: Option<i32> = Some(42);
let no_number: Option<i32> = None;
```
- `Option<T>`是个枚举类型，定义为：
    ```rs
    pub enum Option<T> {
        None,       // 没有值
        Some(T),    // 有值，并把值包在 Some 里
    }
    ```
- 不能将`T`赋值给`Option<T>`
- `Some(42)`表示创建一个Some，里面装着42
- `None`是`Option<T>`枚举的一个值（编译器默认设置其可以全局直接使用）

### option提取
- 法一：`if let`或`while let`结构，判断+提取
    ```rs
    if let Some(item) = x {
        println!("{}", item);
    }
    ```
- 强制提取、崩溃兜底、给默认值
    ```rs
    let item = x.unwrap();        // None 时会 panic（无原因）
    let item = x.expect("must have value"); // None 时会 panic（带上panic原因）
    let item = x.unwrap_or(0);    // None 时返回默认值
    ```
- match
    ```rs
    let item = match x {
        Some(v) => v,
        None    => return,   // 或 break、continue、panic!
    };
    ```
- 错误例子：
    ```rs
    let x: Option<i32> = maybe_negative();   // 可能是 None
    let Some(item) = x;                      // 编译失败，不能这么写（未处理None的情况）
    ```
