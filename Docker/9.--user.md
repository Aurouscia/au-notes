# --user 的作用

可用来控制容器内进程所使用的 UID 和 GID，例如 `--user 33:1000`

## 使用记录

容器 rootful （UID 和 GID 直接映射到宿主机）

容器内的用户名为 www-data，使用 id 查看发现它的 UID 和 GID 分别为 33 和 33

容器需要通过 bind mount 访问宿主机上的特定目录，目录属于宿主机用户 www(1000) 和组 www(1000)

方案：

- 启动容器时指定 UID 和 GID `--user 33:1000`
- 在宿主机新建 www-data 用户，UID 为 33（用来对应容器内的 UID）
- 将新建的 www-data(33) 用户加入 www(1000) 组
- 将目标目录设为 775 （允许同组人任何操作）

效果：

- 容器内进程仍以 UID 33 运行，权限与原本 www-data 等效
- 容器的操作对于宿主机来说就是 33:1000，可以对属于 1000 组的 775 目录进行写入操作
