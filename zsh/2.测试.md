# 测试
注：本语法zsh和bash均支持

## 单中括号\[ ... \]
该语法是兼容POSIX老语法，存在一些限制，但兼容性最好
```bash
# 例1：字符串相等
name="root"
if [ "$name" = "root" ]; then
  echo "当前用户是 root"
fi

# 例2：字符串非空
file="$1"
if [ -n "$file" ]; then
  echo "提供了文件名参数"
fi

# 例3：数字比较（只能用 -eq / -ne / -lt / -le / -gt / -ge）
count=5
if [ "$count" -gt 0 ]; then
  echo "count 是正数"
fi

# 例4：文件存在且是普通文件
if [ -f "/etc/passwd" ]; then
  echo "passwd 文件存在"
fi

# 例5：目录存在且可执行（可进入）
if [ -d "/tmp" ] && [ -x "/tmp" ]; then
  echo "/tmp 存在且可进入"
fi

# 例6：文件是否存在
if [ -s "$HOME/.bashrc" ]; then
    echo ".bashrc 里有内容"
else
    echo ".bashrc 不存在或是空文件"
fi

# 例7：组合条件（注意括号要转义，运算符用 -a -o），-a 表示逻辑与，-o 表示逻辑或
age=25
if [ "$age" -ge 18 -a "$age" -lt 65 ]; then
  echo "适龄工作人口"
fi
```

## 双中括号\[\[ ... \]\]
现代shell的升级版，条件判断优先使用双中括号
- 支持正则表达式：`[[ $str =~ ^[a-zA-Z]+$ ]]`（使用`=~`表示正则匹配）
- 支持glob模式匹配：`[[ $str == *.txt ]]`（`==`和`!=`都可以）

## 等号的区别
- `=`：`单中括号`只能使用`=`，表示两侧相等
- `==`：`双中括号`使用`=`或`==`都行，表示两侧相等，算数运算`$((...))`中只能使用`==`
- `!=`：各种环境都可以使用，含义相同
- `=~`：`双中括号`中，可以使用`=~`表示正则匹配，如`[[ $str =~ ^[a-zA-Z]+$ ]]`表示`$str`是否是纯字母 

## 例子
### 字符串比较
```bash
# 字符串相等比较
[[ $name != $my_name ]]
[[ $email != $my_email ]]

# 字符串模式匹配
[[ $in_now == $need16_prefix || $in_now == $need16_prefix/* ]]
[[ $in_old == $need16_prefix || $in_old == $need16_prefix/* ]]
```

### 文件测试
```bash
# 测试文件是否存在且非空
[ -s "$nvm_sh" ]
[ -s "$nvm_completion" ]

# 测试目录是否存在
[[ -d .git ]]
```

### 复合条件
```bash
# 逻辑或
[[ $in_now == $need16_prefix || $in_now == $need16_prefix/* ]]

# 逻辑与（隐式）
[[ $PWD == /Users/au/my-projects* && -d .git ]]
```
