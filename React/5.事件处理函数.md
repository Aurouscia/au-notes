# 事件处理函数
```jsx
export default function Button() {
  function handleClick() {
    alert('你点击了我！');
  }

  return (
    <button onClick={handleClick}>
      点我
    </button>
  );
}
```

## 与vue的不同之处
onClick={}内，一定不能写括号，否则会在渲染时立即执行。  
这是由于大括号{}内的代码都会立即执行(解析表达式)，而不是在事件触发时执行。
```jsx
//正确写法1：
<button onClick={handleClick}>点我</button>
//正确写法2：
<button onClick={()=>handleClick()}>点我</button>
//错误写法：
<button onClick={handleClick()}>点我</button>
```

## 自定义事件
```jsx
function Button({ onSmash, children }) {
  return (
    <button onClick={onSmash}>
      {children}
    </button>
  );
}
```

## 捕获
在事件属性后加上Capture（是html原版支持的）即onClickCapture={}，可以将事件捕获到父元素上  
（即使子组件e.stopPropagation()了，也会捕获到，比子组件的onClick先执行）  
注：每个事件分3个阶段处理
1. 它向下传播，调用所有的 onClickCapture 处理函数
2. 它执行被点击元素的 onClick 处理函数
3. 它向上传播，调用所有的 onClick 处理函数

## 副作用
事件处理函数可以有副作用（更改状态）  
不需要遵循“React组件必须是纯函数”的规则
