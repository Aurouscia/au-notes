# 错题集：指针

## 一、指针变量声明语法

**错误题目**：声明一个指向 `int` 类型的指针变量，应该写为________。

**错误答案**：`var *int`

**正确答案**：`var p *int`

**知识点**：
- 指针变量声明格式：`var 变量名 *类型`
- `*int` 是类型，`p` 是变量名，不能写成 `var *int`

## 二、指针赋值后的指向变化

**错误题目**：
```go
a := 1
b := 2
p1 := &a    // p1 指向 a
p2 := &b    // p2 指向 b
p1 = p2     // p1 现在指向 b
*p1 = 100   // 修改 p1 指向的值（即 b）
fmt.Println(a)  // ?
fmt.Println(b)  // ?
```

**错误答案**：a = 100, b = 100

**正确答案**：**a = 1, b = 100**

**分析**：
1. `p1 := &a` → p1 指向 a
2. `p2 := &b` → p2 指向 b
3. `p1 = p2` → **p1 被重新赋值为 p2 的值**，现在 p1 也指向 b
4. `*p1 = 100` → 修改 p1 指向的值（即 b），所以 b = 100
5. a 从未被修改，所以 a = 1

**图示**：
```
初始状态：          p1 = p2 后：
p1 → a(1)          p1 ─┐
p2 → b(2)          p2 ─┼→ b(2)
                   a(1)单独存在
```

## 三、返回局部变量指针（逃逸分析）

**错误题目**：以下代码有什么问题？请说明原因。
```go
func getPointer() *int {
    x := 10
    return &x
}
```

**错误答案**：返回局部变量的指针（逃逸），不推荐这么做

**完整答案**：
- 代码**可以正常运行**，Go 编译器会进行**逃逸分析**，将 x 分配到堆上
- 但这不是好的编程习惯
- 应该避免返回局部变量指针，改用传入指针参数或返回值的写法

**知识点**：
- Go 编译器会自动进行**逃逸分析**（Escape Analysis）
- 如果局部变量的地址被返回，编译器会将其分配到堆上，而不是栈上
- 虽然能运行，但会导致堆内存分配，增加 GC 压力

**更好的写法**：
```go
// 方式1：返回值
func getValue() int {
    x := 10
    return x
}

// 方式2：传入指针
func setValue(p *int) {
    *p = 10
}
```

## 总结

| 错误类型 | 关键知识点 |
|---------|-----------|
| 指针声明 | `var p *int`，变量名在前，`*类型`在后 |
| 指针赋值 | 指针变量可以被重新赋值指向其他地址 |
| 逃逸分析 | Go 允许返回局部变量指针（编译器会处理），但不推荐 |
