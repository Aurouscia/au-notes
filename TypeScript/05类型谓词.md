# 类型谓词
`is`关键字用来写类型谓词（type predicate），
它把“运行时布尔返回值”和“编译时类型收窄”绑定在一起

注意：`is`和`as`一样，会被ts无条件信任，ts无法在编译期检查`is`的正确性

## 示例
### 例子1
```ts
function isString(x: unknown): x is string {
  return typeof x === 'string';
}

const v: unknown = 'hello';
v.toUpperCase(); // 无法通过ts类型检查
if (isString(v)) {
    v.toUpperCase(); // ✅ 由于isString的返回值类型，v被收窄成string
}
```

### 例子2
`for…in` 的设计目标是“遍历对象上所有可枚举属性”（包括原型链属性，以及不存在于T的其他属性）  
而ts无法保证这些属性一定存在于“T的声明键集合”里  
所以不能把`for…in`的`key`直接缩成`keyof T`

```ts
const obj = { a: 1, b: 2 };

function isKeyOf<T extends object>(
    k: PropertyKey,
    o: T
): k is keyof T {
  return k in o;
}

for (const key in obj) {
    console.log(obj[key]); // 直接写会出问题，obj不具有string的索引签名
    if (isKeyOf(key, obj)) {
        console.log(obj[key]); // OK，key 被收窄成 'a' | 'b'
    }
}
```

### 例子3
```ts
function hasA(x: unknown): x is { a: string } {
  // 1. 必须是对象且不为 null
  if (typeof x !== 'object' || x === null) return false;

  // 2. 安全地读取属性，再判断类型
  return typeof (x as Record<PropertyKey, unknown>).a === 'string';
}
```
