## 一、填空题

1. Go Module 是从 Go ______ 版本开始引入的依赖管理系统。 
    - 1.11

2. `go.mod` 文件的作用是 ______，而 `go.sum` 文件的作用是 ______。 
    - 记录当前模块的名称、依赖和go版本  
    - 记录各个依赖的校验和

3. 初始化一个新模块的命令是 ______。
    - go mod init

4. 在开发过程中，当我们引入了新的依赖包后，应该运行 ______ 命令来整理依赖。
    - go mod tidy

## 二、判断题（正确写"对"，错误写"错"）

1. `go mod download` 会修改 go.mod 文件，添加新的依赖项。
    - 错

2. 如果删除了 `go.sum` 文件，项目就无法编译了。
    - 错

3. 本地开发项目可以使用简单的模块名如 `myapp`，而开源项目建议使用 `github.com/username/repo` 这样的路径。
    - 对

4. `go mod tidy` 会删除未使用的依赖项。
    - 对

## 三、简答题

1. 请简述 `go mod download` 和 `go mod tidy` 的区别。
    - download 仅会下载依赖，不会改变 go.mod 和 go.sum 文件
    - tidy 会根据代码添加需要的依赖项，删去未使用的
    - download 一般用于部署，而 tidy 一般用于开发

2. 假设你删除了 `go.sum` 文件，如何恢复它？请写出具体命令。
    - go mod download 和 go mod tidy 均可
