【文件名必须与类名一致】
写好的脚本必须附加到物体上才执行
附加到游戏物体的脚本类必须从MonoBehaviour类继承
写属性也没用，因为编辑器里没法看到
【不要在脚本里写构造函数】

一个脚本就是一个类，附加到物体上之后就会创建一个对象
Update方法每帧调用一次，不用就删掉以节省性能

D:\Unity\Editor\Data\Resources\ScriptTemplates

【【脚本生命周期】】
也叫：必然事件，消息

因为：脚本里写的是类(默认数据)，挂到物体上是对象，组件栏修改的是对象的数据

****************初始阶段*****************
【Awake()】【创建游戏对象时】一次
可用来判断是否执行此脚本this.enable=true;
可用来初始化(new 数组/随机数生成器)
【Start()】【脚本启用时执行】一次
所有物体的Awake都执行完了之后，再执行所有物体的Start
也用来初始化
【OnEnable()】【脚本启用时执行】每次
****************物理阶段*****************
不可以在这些频繁调用的函数里new任何对象否则会产生大量垃圾
做的事情尽量少
【FixedUpdate()】
每隔固定时间做一次(0.02秒)不会受到渲染时间的影响
适用于对物体做物理操作，让物体运动
【Update()】
每次渲染场景执行一次，执行间隔不固定
适用于处理游戏逻辑
【LateUpdate()】
在Update执行之后执行，适用于跟随逻辑，其实是同一帧
****************输入阶段****************

鼠标移入碰撞箱时
【OnMouseOver】
鼠标经过碰撞箱时
【OnMouseExit】
鼠标离开碰撞箱时
【OnMouseDown】
在碰撞箱按下时
【OnMouseUp】
在碰撞箱松开时

【OnBecameInvisible】仅一帧
当在任何相机上都不可见时调用

【Component类提供了查找(当前，后代，先辈)组件的功能】
想访问某个游戏物体的tag，可以拿任意一个组件的引用点出来
因为：Component类有个tag的属性
而那些组件都是Component类的子类，所以可以访问到
在脚本里可：
this.transform.position = new Vector3(1, 1, 1);
有些组件没法直接访问，要用
GetComponent<组件名>()
var allComp=this.GetComponent<Component>()可以返回所有组件
foreach(var item in allComp)
{print(item.GetType());}

if (GUILayout.Button("Color"))【把自己和儿子都变红】
{//获取后代物体的指定类型组件(从自己开始)
      var allComp = this.【GetComponentsInChildren】<MeshRenderer>();
      foreach(var item in allComp)
      {
           item.material.color = Color.red;
      }
}
GetComponentInChildren<>:找一个
例如：在一个人物中找到audioSource

【显示为Enumerable的类可以这么遍历】
