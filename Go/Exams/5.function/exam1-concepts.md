# 函数基础 - 概念题

## 一、填空题

1. Go 语言使用关键字 ______ 定义函数。
    - func

2. 当多个相邻参数类型相同时，可以只写一次类型，例如 `func add(a, b int) int` 中，参数 `a` 和 `b` 的类型都是 ______。
    - int

3. 函数返回多个值时，返回值列表需要用 ______ 包裹。
    - ()

4. 在命名返回值的情况下，`return` 语句可以省略返回值变量，这种写法称为 ______ 返回。
    - 裸

5. 如果只想获取函数的部分返回值，可以使用 ______ 来忽略不需要的返回值。
    - 下划线

## 二、判断题（请在每题后写出判断结果和理由）

1. Go 语言函数默认使用引用传递，函数内对参数的修改会影响原变量。
    - 错，默认值传递

2. 以下函数定义是合法的：
   ```go
   func calculate(x int, y int) (sum int, diff int) {
       sum = x + y
       diff = x - y
       return
   }
   ```
    - 对

3. 命名返回值在函数开始执行时会被初始化为对应类型的零值。
    - 对

4. 函数必须至少有一个返回值。
    - 错，可以有裸返回

5. 以下代码中，`main` 函数打印的 `x` 和 `y` 值会被交换：
   ```go
   func swap(a, b int) {
       a, b = b, a
   }
   
   func main() {
       x, y := 1, 2
       swap(x, y)
       fmt.Println(x, y)
   }
   ```
    - 错，仅仅改了函数内的局部变量，没有改外面的原值

## 三、简答题

1. 请说明值传递和指针传递的区别，并分别给出适用场景。
    - 值传递无法改变原值，指针传递可以
    - 例如：打印某值的方法使用值传递，修改某值的方法使用指针传递

2. 命名返回值有什么优点和缺点？什么情况下建议使用命名返回值？
    - 命名返回值可以简化 return 语句，但在大函数中会破坏可读性
    - 小函数中可以使用命名返回值

3. 以下代码的输出是什么？请解释原因。
   ```go
   func modify(s []int) {
       s[0] = 100
   }
   
   func main() {
       arr := []int{1, 2, 3}
       modify(arr)
       fmt.Println(arr[0])
   }
   ```
    - 100，因为切片是引用类型，不会复制进去
