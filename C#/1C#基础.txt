CLR会把申请的空间进行逻辑划分：
栈区：小，快。分配的区域叫栈帧，
用于存储正在执行的方法和其中的参数和变量，用完就释放。
堆区：大，慢。用于存储引用类型。

【局部变量】：定义在方法内部的变量
1.没有初始值，必须设定初始值才能使用。
2.方法被调用时存在栈内，方法结束时消失。
【方法执行在栈中，所以方法中声明的变量都在栈中】
int a=1;
int[] arr=new int[]{2,3};
【值类型直接存储值，a在栈上(方法内)1也在栈上(方法内)】
arr也在栈上(方法内)，new int[2]{2,3}在堆上(new指在堆中开辟一块空间)
【arr是堆上2,3的内存地址】
int b=a时，把a的值(1)给了b

【装箱】：值类型隐式转换为object类型
int a=1;
object o=a;
在堆里开了一块空间，分为三小块：
(int，同步块索引，类型对象指针)
然后把1放进int
(栈中)o指向(堆中)1的位置
//////
拆箱：object类型显式转换为值类型
int b=(int)o;
检查类型是否相同，
o指向的int给b
//////
形参是object，实参是值时就装箱
可以通过重载/泛型来替代
例子：
int num=100;
string str=num.ToString();没有拆装箱
但int num=100;
string str=""+num;
相当于str=string.Concat("",num);字符串拼接函数(参数都是object)有拆装箱
改成str=""+num.ToString;就没有了
对性能影响不大，但面试会问(tmd)

【字符串池】写代码时，源代码里的字符串常量，
会进入字符串池存储。赋值时会在池里寻找，直接返回已有的引用，节省空间
str2="456";
并没有修改字符串，只修改了引用：
【字符串的不可变性】(为了避免分配的连续内存溢出)
同理：object o1=1;
o1=2.0;
o1="ok";
o1=true;
每次修改都是新开辟空间存储数据，替换引用
因为字节数不一样
所以：频繁拼接字符串会产生很多垃圾
string strNum ="";
for(int i=0;i<10;i++){
    strNum+=strNum+i.ToString();
}
Console.WriteLine(strNum);不可以这么写
应该：创建【可变字符串】
（一个叫StringBuilder的类，并不是字符串）
StringBuilder builder=new StringBuilder((可以不填)字符数);
指定字符数后超了也不要紧，会自动另外开一个更大的空间，并替换引用
(还是产生了垃圾)
for(int i=0;i<10;i++){
    builder.Append(i);
}
string result=builder.ToString();
这样可以避免产生垃圾
builder.Insert(0,"abc");可以直接给builder的0位置插入字符串
builder.Replace();
builder.Remove();
区别于：
s1=s1.Insert(0,"abc");必须用返回值重新赋值，替换引用
