# 输出重定向
- `>`：将命令输出重定向到文件
- `>>`：将命令输出追加到文件
- `1>`：将标准输出重定向到文件
- `2>`：将标准错误输出重定向到文件
- `2>&1`：将标准错误输出重定向到标准输出
- `&>`：将标准输出和标准错误输出都重定向到文件（`2>&1`的语法糖）
- `&>>`：将标准输出和标准错误输出都追加到文件

## 例子
```bash
# 将错误输出重定向到/dev/null（丢弃）
git config --get user.name 2>/dev/null
```

## 注意顺序
`cmd 1>all.log 2>&1`：先将标准输出重定向到`all.log`，然后将标准错误输出重定向到标准输出，即标准输出和标准错误输出都重定向到`all.log`
`cmd 2>&1 1>all.log`：先将标准错误输出重定向到标准输出，然后将标准输出重定向到`all.log`，此时错误输出并不会被重定向到`all.log`，因为标准错误输出已经先被重定向到原先的标准输出（通常是终端），不会跟随`1`而变动
