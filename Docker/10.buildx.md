# buildx

buildx 可以让 docker 构建多种架构的镜像

## 管理 buildx 实例

一个 docker 客户端可以有多个 buildx 实例

- docker buildx ls
- docker buildx create --name xxx
- docker buildx create --name xxx --use
    - --use 创建后立即设为当前默认使用的 buildx 实例
- docker buildx rm xxx

创建 buildx 实例时，会需要用到 moby/buildkit:buildx-stable-1 镜像，建议提前通过科学上网获取该镜像

## 使用 buildx

docker buildx build --builder xxx --platform linux/amd64,linux/arm64 -t your-registry/fickit-backend --push .

- --builder 指定 buildx 实例
- --platform 指定要构建的架构
- --push 立即推送
- . Dockerfile位置（当前目录）

## 立即推送

受限于 docker 客户端的存储方式，无法存储同一个镜像的多种架构，只能指定 --push 立即推送去 registry

## insecure-registry

如果 registry 没有 HTTPS 证书，需要显式允许连接

buildx 与 docker 本体的配置是分开的，无法继承 docker desktop 的 insecure-registries 配置

- 在 ~/.docker/buildx 新建文件 yyy.toml
- docker buildx create --use --name xxx --config ~/.docker/buildx/yyy.toml --bootstrap
    - --config 指定创建时使用的配置文件
    - --bootstrap 立即启动
