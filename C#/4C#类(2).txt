【static静态】
能修饰：成员变量，构造函数，方法，类
没static的：就叫实例xx
public static int instanceCount;
Console.WriteLine(【Student.instanceCount】);用类名点

静态成员变量属于类，类被加载时初始化，且只有一份
类：出现类名时即“加载”
存储在“静态区”，所有对象共享
实例成员变量属于对象，在每个对象被new时初始化，每个对象一份

在没有对象时就要访问的成员

【静态构造函数】
static Student(){ }
实例构造函数：初始化类的实例数据成员
（其实放外面也差不多，但是初始化放在构造函数里面比较规范）
	private Random random;
	public Student(){random=new Random()}
静态构造函数：初始化类的静态数据成员，执行时机：【仅在类加载时调用一次】
静态的数据成员最好是这里初始化
错误提示："非静态字段要求对象的引用"：即静态代码块只能访问静态成员
【类内的方法】
并不在对象的内存空间里，而是所有对象共用，
对象调用方法时把自己的引用传给方法，然后栈上会有个栈帧来运行这个方法
stu03.Fun2();其实自动传递了stu03的引用作为参数
然后顺着引用找到对象内的方法
所以静态的方法（由类点出来）
没有传递任何对象引用，无法运算实例数据

Math math = new Math();太麻烦了

【结构体】
与类差不多，但是类是引用类型，结构体是值类型，
存取速度快一些，适用于大量的小型相关变量（位置，颜色）
缺点：变量多时会占用大量栈空间
