# 使容器开机自启动

推荐做法是使用 systemd 管理

## 1. 编写 .service 文件

```ini
[Unit]
# Description: 服务的描述信息，显示在 systemctl status 中
Description=CRMEB Redis Container

# After: 指定该服务在哪些目标之后启动
# network.target 表示网络服务启动后再启动本服务
After=network.target
# 也可以添加其他依赖，如：
# After=network.target docker.service
# Requires=docker.service

[Service]
# Type: 服务启动类型
# simple - 前台运行（默认），ExecStart 启动的进程为主进程
# forking - 后台运行，父进程退出，子进程继续运行
# oneshot - 一次性任务，执行完即退出
# notify - 服务启动后会发送通知信号
Type=simple

# Restart: 服务退出后的重启策略
# no - 不重启（默认）
# always - 总是重启
# on-success - 正常退出时重启（退出码 0）
# on-failure - 异常退出时重启（非 0 退出码、信号终止、超时）
# on-abnormal - 被信号终止或超时时重启
# on-abort - 被信号终止时重启
Restart=always

# RestartSec: 重启前等待的秒数，避免过于频繁重启
RestartSec=5

# StartLimitIntervalSec: 在多长时间内限制重启次数（0 表示不限制）
# StartLimitBurst: 在给定时间间隔内最多重启次数
StartLimitIntervalSec=60
StartLimitBurst=3

# TimeoutStartSec: 启动服务超时时间
TimeoutStartSec=60

# TimeoutStopSec: 停止服务超时时间（等待容器优雅停止的时间）
TimeoutStopSec=40

# User/Group: 以哪个用户/组身份运行服务（rootless 模式常用）
# User=www-data
# Group=www-data

# WorkingDirectory: 设置工作目录
# WorkingDirectory=/opt/myapp

# Environment: 设置环境变量
# Environment="KEY=value"
# EnvironmentFile: 从文件加载环境变量
# EnvironmentFile=/etc/default/myapp

# 启动命令 - 使用 podman/docker 运行容器
ExecStart=/usr/bin/podman run \
    --name crmeb-redis \
    # --rm: 容器停止后自动删除容器（注意：不要 -d，会导致服务状态管理失效）
    --rm \
    # --pod: 将容器加入指定的 pod
    --pod crmeb-pod \
    # -v: 挂载数据卷，:z 表示为容器设置 SELinux 标签
    -v /www/wwwroot/kafei-redis-data:/data:z \
    # 使用的镜像
    redis:7-alpine \
    # 容器启动命令
    redis-server --appendonly yes

# ExecStartPre: 启动前执行的命令（可多次使用，按顺序执行）
# ExecStartPre=-/usr/bin/podman rm -f crmeb-redis
# 前面的 "-" 表示命令失败也不影响后续操作

# ExecStartPost: 启动后执行的命令
# ExecStartPost=/usr/bin/podman ps -f name=crmeb-redis

# ExecStop: 停止服务的命令
# -t 30: 等待 30 秒让容器优雅停止，超时后强制停止
ExecStop=/usr/bin/podman stop -t 30 crmeb-redis

# ExecStopPost: 停止后执行的命令
# 容器停止后删除容器（配合 --rm 使用，确保清理）
ExecStopPost=/usr/bin/podman rm crmeb-redis

# ExecReload: 重新加载配置（如支持热重载的服务）
# ExecReload=/usr/bin/podman kill -s HUP crmeb-redis

# KillMode: 如何终止服务进程
# control-group - 终止控制组中的所有进程（默认）
# process - 仅终止主进程
# mixed - 先发送 SIGTERM 给主进程，再发送 SIGKILL 给控制组
# none - 不发送信号
KillMode=mixed

[Install]
# WantedBy: 启用服务时，创建符号链接到哪个 target
# multi-user.target - 多用户命令行模式（无图形界面）
# graphical.target - 图形界面模式
# default.target - 系统默认目标
WantedBy=multi-user.target

# Also: 同时启用/禁用的其他服务
# Also=container-my-nginx.service

# Alias: 服务别名，可以用别名来操作服务
# Alias=redis-container.service
```

## 2. 启用该服务

> 注意：如果是用户自己的服务，应使用 `systemctl --user`

```bash
# 重新加载 systemd 配置（修改 .service 文件后必须执行）
sudo systemctl daemon-reload

# 设置开机自启动，并立即启动（--now）
sudo systemctl enable --now container-my-nginx.service

# 仅设置开机自启动，不立即启动
sudo systemctl enable container-my-nginx.service

# 立即启动，但不设置开机自启动
sudo systemctl start container-my-nginx.service
```

## 3. 常用管理命令

```bash
# 查看容器状态
podman ps

# 查看 Systemd 服务状态
sudo systemctl status container-my-nginx.service

# 查看是否已启用开机自启动
sudo systemctl is-enabled container-my-nginx.service

# 启动/停止/重启服务
sudo systemctl start container-my-nginx.service
sudo systemctl stop container-my-nginx.service
sudo systemctl restart container-my-nginx.service

# 重新加载服务配置（不重启）
sudo systemctl reload container-my-nginx.service

# 禁用开机自启动
sudo systemctl disable container-my-nginx.service

# 禁用并立即停止
sudo systemctl disable --now container-my-nginx.service

# 查看服务日志
sudo journalctl -u container-my-nginx.service

# 实时跟踪日志
sudo journalctl -u container-my-nginx.service -f

# 查看最近的日志
sudo journalctl -u container-my-nginx.service --since "10 minutes ago"
```

## 4. 文件存放位置

```bash
# 系统级服务（需要 root 权限）
/etc/systemd/system/           # 自定义服务
/usr/lib/systemd/system/       # 软件包安装的服务
/run/systemd/system/           # 运行时生成的服务

# 用户级服务（无需 root，使用 systemctl --user）
~/.config/systemd/user/        # 用户自定义服务
/etc/systemd/user/             # 系统范围的用户服务
/usr/lib/systemd/user/         # 软件包安装的用户服务
```

## 5. 注意事项

### Rootful vs Rootless

对 podman 来说，systemctl 启动服务是否 sudo，决定容器是 rootful 还是 rootless：

| 命令 | 模式 | 说明 |
|------|------|------|
| `systemctl --user` | Rootless | 以普通用户身份运行容器 |
| `sudo systemctl` | Rootful | 以 root 身份运行容器 |

### 常用最佳实践

1. **使用 `--rm`**：配合 `ExecStopPost` 确保容器停止后自动清理
2. **设置超时**：`TimeoutStopSec` 应大于 `podman stop` 的 `-t` 时间
3. **限制重启**：使用 `StartLimitIntervalSec` 和 `StartLimitBurst` 防止无限重启
4. **日志管理**：考虑添加日志轮转配置
5. **依赖管理**：使用 `After=` 和 `Requires=` 确保服务按正确顺序启动

### 示例：Docker 版本

```ini
[Unit]
Description=My Nginx Container
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
Restart=always
RestartSec=5

ExecStartPre=-/usr/bin/docker rm -f my-nginx

ExecStart=/usr/bin/docker run \
    --name my-nginx \
    --rm \
    -p 80:80 \
    -v /opt/nginx/html:/usr/share/nginx/html:ro \
    nginx:alpine

ExecStop=/usr/bin/docker stop -t 30 my-nginx
ExecStopPost=/usr/bin/docker rm my-nginx

[Install]
WantedBy=multi-user.target
```
