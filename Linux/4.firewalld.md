# firewalld

firewalld 就是 iptables 外面包了一层

如果启用了 firewalld，不建议编辑 iptables，会打架（互相覆盖）应该直接使用 firewalld 管理

## zone

firewalld 可以通过 zone 来管理什么来源能访问什么端口，例如：

```bash
# 创建自定义区域（zone）用于数据库访问
sudo firewall-cmd --permanent --new-zone=mysql-access

# 将信任的网段绑定到该区域
sudo firewall-cmd --permanent --zone=mysql-access --add-source=10.88.0.0/16
sudo firewall-cmd --permanent --zone=mysql-access --add-source=127.0.0.1/8

# 在该区域开放 MySQL
sudo firewall-cmd --permanent --zone=mysql-access --add-port=3306/tcp

# 从默认区域移除 MySQL（确保其他区域不能访问）
sudo firewall-cmd --permanent --zone=public --remove-port=3306/tcp 2>/dev/null
sudo firewall-cmd --permanent --zone=public --remove-service=mysql 2>/dev/null

# 重载（--permanent 后必须）
sudo firewall-cmd --reload

# 查看区域配置
sudo firewall-cmd --zone=mysql-access --list-all
```

## 查看状态

```bash
# 查看 firewalld 运行状态
sudo systemctl status firewalld

# 查看默认区域
sudo firewall-cmd --get-default-zone

# 查看当前生效的所有规则
sudo firewall-cmd --list-all

# 查看特定区域的规则
sudo firewall-cmd --zone=public --list-all

# 查看所有区域配置
sudo firewall-cmd --get-active-zones
sudo firewall-cmd --list-all-zones

# 查看富规则（rich rules）
sudo firewall-cmd --list-rich-rules

# 查看服务定义
sudo firewall-cmd --get-services | grep mysql
```
