# 函数类型表达式
```ts
const fn: (a: string) => void
```
表示：一个接受字符串参数且没有返回值的函数对象

注意一定要写参数名称，如果写`(string)=>void`那就表示一个名为string的参数了
```ts
function greeter(fn: (a: string) => void) {
  fn("Hello, World");
}
function printToConsole(s: string) {
  console.log(s);
}
greeter(printToConsole);
```

# 调用签名
如果想让函数本身有属性值，那么需要定义一个可以调用的对象类型
```ts
type DescribableFunction = {
  description: string;
  (someArg: number): boolean;
};
function doSomething(fn: DescribableFunction) {
  console.log(fn.description + " returned " + fn(6));
}
 
function myFunc(someArg: number) {
  return someArg > 3;
}
myFunc.description = "default description";
 
doSomething(myFunc);
```

## 构造签名
前面加上new
```ts
type SomeConstructor = {
  new (s: string): SomeObject;
};
function fn(ctor: SomeConstructor) {
  return new ctor("hello");
}
```

# 泛型
有时候输入和输出参数不确定但是互相关联时，可以使用泛型指定，例如取出数组的第一个元素
```ts
function firstElement<T>(arr: T[]): T | undefined {
  return arr[0];
}
function map<Input, Output>(arr: Input[], func: (arg: Input) => Output): Output[] {
  return arr.map(func);
}
// Parameter 'n' is of type 'string'
// 'parsed' is of type 'number[]'
const parsed = map(["1", "2", "3"], (n) => parseInt(n));
```

## 泛型的约束
使用extends关键字进行类型约束

限定：处理的类型T必须有一个名为length，类型为number的属性，否则报错
```ts
function longest<T extends { length: number }>(a: T, b: T) {
  if (a.length >= b.length) {
    return a;
  } else {
    return b;
  }
}
```

### 注意
类型和类型的约束并不能互相代替作为返回值

调用泛型函数时可以让ts自动识别类型，识别出错时也可以手动指定
```ts
const arr = combine<string | number>([1, 2, 3], ["hello"]);
```

如果可以的话，尽量直接规定参数类型，而不是给泛型加约束（可能推断失败）

如果可以的话，尽量用数量更少的泛型约束（以免函数难以调用）

如果泛型的类型参数只出现了一次，那就是没用的。

# 可选参数
## 问号标记
```ts
function f(x?:number){}
```
如果上述这样写，那么x在函数内就是number|undefined类型

调用函数时，可以给一个参数，也可以不给
## 给默认值
```ts
function f(x=10)
```
调用函数时，可以给一个参数，也可以不给，不给就会自动填10

# 重载
ts里的重载不一样。

写法是先写两个以上的函数签名（表示它可能被调用的方式）最后跟上有函数体的函数实现。
```ts
//函数签名
function makeDate(timestamp: number): Date;
//函数签名
function makeDate(m: number, d: number, y: number): Date;
//函数实现
function makeDate(mOrTimestamp: number, d?: number, y?: number): Date {
  if (d !== undefined && y !== undefined) {
    return new Date(y, mOrTimestamp, d);
  } else {
    return new Date(mOrTimestamp);
  }
}
const d1 = makeDate(12345678);
const d2 = makeDate(5, 5, 5);
```
函数签名和实现的"各个位置的参数和返回值"必须类型都对的上

注意：能使用联合类型就别用重载

# 对this的类型指定
```ts
interface DB {
  filterUsers(filter: (this: User) => boolean): User[];
}
 
const db = getDB();
const admins = db.filterUsers(function (this: User) {
  return this.admin;
});
```
此处为了筛选用户把一个回调函数传给了User对象，所以让此处的this指的是一个User对象

注意使用这种写法时，必须使用function关键字来定义那个回调函数，不能使用箭头函数（因为箭头函数没有自己的this）

# 其他类型
## void
void表示一个函数没有返回值，如果读取返回值那就是undefined，但void和undefined是两个不同的类型。

## object
object表示原始类型（string, number, bigint, boolean, symbol, null, undefined）之外的其他值。

注意区别于首字母大写的Object，是两码事。

函数对象也会被认为是object类型

## unknown
unknown和any表示的意思一样，但**更安全**，因为它不允许读取任何属性。表示这个对象需要小心对待。
```ts
function f1(a: any) {
  a.b(); // OK
}
function f2(a: unknown) {
  a.b(); //报错
}
```
any可以隐式转换为unknown

## never
never表示一个不可能被读取到的对象。例如一个必定会终结程序运行的函数的返回值，或者被收窄到没有的联合类型。

## Function
表示一个可以接受任何数量和类型的参数，返回any的玩意。不要用，不安全。

## rest参数
用三个点表示任意数量的参数输入，类型必须是T\[ \](某种数组)
```ts
function multiply(n: number, ...m: number[]) {
  return m.map((x) => n * x);
}
// 'a' gets value [10, 20, 30, 40]
const a = multiply(10, 1, 2, 3, 4);
```
也可以：用一个可遍历的对象提供参数
```ts
const arr1 = [1, 2, 3];
const arr2 = [4, 5, 6];
arr1.push(...arr2);
```

注意：ts默认认为数组并非不可变，所以
```ts
// Inferred type is number[] -- "an array with zero or more numbers",
// not specifically two numbers
const args = [8, 5];
const angle = Math.atan2(...args);
```
报错，但是如果使用as const语法将其改成字面量类型，就没问题了
```ts
// Inferred as 2-length tuple
const args = [8, 5] as const;
// OK
const angle = Math.atan2(...args);
```

## 参数拆解
js支持传入一个对象，将其每一个属性作为一个参数

这种时候想要自定义
```ts
function sum({ a, b, c }: { a: number; b: number; c: number }) {
  console.log(a + b + c);
}
```
后面那个类型定义换成一个named type也行
```ts
// Same as prior example
type ABC = { a: number; b: number; c: number };
function sum({ a, b, c }: ABC) {
  console.log(a + b + c);
}
```

注意：参数拆解也可以定默认值

注意写法是等于号而不是冒号（与js里的对象定义搞混了）

```ts
type ABC = { a: number; b?: number; c?: number }; //注意这里有被标记为可能缺少的值
function sum({ a, b = 20, c = 30 }: ABC) { //这里的实现给了默认值
  console.log(a + b + c);
}
```
