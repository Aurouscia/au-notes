# 需求

服务器上有新旧两个应用实例，分别位于两个端口

需要将特定流量（开发/测试人员）分配到新实例，同时不影响旧实例的运作

## map 块

在 server 块前添加：

```conf
map $cookie_backend $swoole_backend {
    default 127.0.0.1:20699;
    test    127.0.0.1:20700;
}
```

含义：对于 cookie 中的 backend 字段
- 如果值为 test，则将 $swoole_backend 设为 127.0.0.1:20700
- 如果无值或是其他值，则将 $swoole_backend 设为 127.0.0.1:20699

## location 块

在 location 块内修改 proxy_pass 的值：

```conf
proxy_pass http://$swoole_backend;
```

使用之前设置的变量值作为代理目标，实现不同的流量分配

## 使用

在浏览器开发人员工具手动添加 cookie，键为 backend，值为 test，即可使用新实例
