# 进制转换
- 二进制转十进制
    - 从右往左，每个数乘以2的(位数-1)次方，然后相加
    - 例如：110.1转十进制
        - (1 * 2^2) + (1 * 2^1) + (0 * 2^0) + (1 * 2^-1) = 6.25
- 二进制转八进制
    从小数点起，每三位为一组，不足三位的补0，每组对应一个八进制数
    - 100 : 4
    - 101 : 5
    - 111 : 7
    - 001 011 010 . 010 : 132.2
- 二进制转十六进制
    从小数点起，每四位为一组，不足四位的补0，每组对应一个十六进制数
    - 1000 : 8
    - 1001 : 9
    - 1111 : F
    - 0011 0110 . 0101 1100 : 36.5C
- 十进制转二进制，例：175.125
    - **整数部分：除二取余，最后从下往上取：最后一个结果+所有余数**
        | 除数 | 被除数 | 余（上一排的） |
        | ---- | ----- | ---- |
        | 2    |  175  | /  |
        | 2    |  87   | 1  |
        | 2    |  43   | 1  |
        | 2    |  21   | 1  |
        | 2    |  10   | 1  |
        | 2    |  5    | 0  |
        | 2    |  2    | 1  |
        | /    |  1    | 0  | 
        结果为：10101111
    - **小数部分：乘基数取整，最后从上往下取**
        0.125 * 2 = 0.25 （整数部分为0）
        0.25 * 2 = 0.5（整数部分为0）
        0.5 * 2 = 1（整数部分为1）
        结果为：0.001
        - 二进制的0.1代表2^(-1) 即 0.5
        - 二进制的0.01代表2^(-2) 即 0.25
- 二进制的运算
    - 加法：逢二进一
    - 减法：借一当二
    - 乘法：异或

# 码值
- 原码：最高位表示正负（0正1负），其他的表示数值
    - +5: 0 000 0101, -5: 1 000 0101
    - +0: 0 000 0000, -0: 1 000 0000 （正负0不同）
    - 缺点：存在正负两个0，不能直接进行加减运算（现代计算机不使用）
- 反码：正数与原码相同，负数是原码按位取反（符号位除外，依然是1）
    - +5: 0 000 0101, -5: 1 111 1010
    - 缺点：依然存在正负两个0，运算时有“循环进位”复杂处理（现代计算机不使用）
- 补码：正数与原码相同，负数是反码的值+1
    - +5: 0 000 0101, -5: 1 111 1011
    - 优点：可以将减法直接转换为加法，无需考虑操作数的符号
        7-5可直接当做7+(-5)表示（最高位自然丢弃）
    - 现代计算机广泛使用
    - 只有一个0，没有+0和-0的区别
    - 原码和反码都无法表示-128、而补码可以
- 移码：*符号位反着（1正0负）*
    - 会定义一个偏移量（8位移码一般为128）
    - 将原值统一加上偏移量后表示
    - 5: (加偏移量后133) 1 000 0101
    - -5: (加偏移量后123) 0 111 1011
    - 0: (加偏移量后128) 1 000 0000
    - 优点：比较大小非常方便

# 浮点数
(-1)^(<符号位>) * (1.<尾数>) * 2^(<指数>)
考点：
- **阶码（指数部分）的位数**决定数值的 表示范围
- **尾数（有效数字）**决定数值的 精度
- **规格化**是值：将尾数的绝对值限定在区间0.5(包括)~1(不包括)

## 浮点数的运算
- 对阶
    - 较小数向较大数靠近
    - 通过**较小数的尾数右移**实现
- 尾数计算
- 结果格式化
- 溢出：相同符号相加，或相反符号相减，才可能溢出

# 校验码
- 奇偶校验
    - 最后加一位，使得整个数据中的**1的个数**保持奇数（或偶数）
        - 单个错误：只能检测出，无法修复
    - 二维奇偶校验，可以检测并修复单个错误
        - 对每一行每一列生成一个奇偶校验位
        - 把奇偶校验位添加为最后一行和最后一列
- 海明校验（汉明码）
    - 对于k位数据，需要r个校验位，使得 `2^r ≥ k+r+1`
    - 校验位放在第：1、2、4、8、16...位
    - 单个错误：不仅能检测，还能纠正
- 循环冗余
    - **采用模2除法（实际上就是异或）**进行计算
    - 一般在信息的传递中使用（只检测不修复）
    - 单个错误：100%检测出
    - 多个错误：大概率检测出
