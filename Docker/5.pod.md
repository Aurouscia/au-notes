# pod

pod 是 podman 独有的一种功能，用于容器之间共享 Network/IPC/PID

可以使用 pod 解决跨容器通信问题（network无法解决时）

## 前提条件

旧版本 Podman 需要手动准备 pause 镜像作为 INFRA（基础设施），创建 pod 时可能会卡住并报错无法获取 xxxx/pause 镜像。新版本 Podman 通常会自动构建本地 infra 镜像，无需手动拉取。

如果遇到 pause 镜像缺失的问题，可以在 registry.aliyuncs.com/google_containers 拉取同名镜像，然后 tag 重命名成它要求的那个，即可创建。

## 使用

容器启动时，使用 `--pod <pod-name>` 指定使用的 pod

pod 内的容器不能有自己的端口映射设置，统一由 pod 管理：

```sh
sudo podman pod create --name xxx -p 1234:2345
```

pod 内的容器共享一个 0-65535 端口范围，可以通过 localhost 互相访问（适用于 redis 和数据库等）

## 有无 sudo 的区别

- 使用 sudo 创建的 pod 是 rootful 的
    - 可以直接通过 127.0.0.1 访问其监听的端口
    - -p 设置端口映射时，可以指定 127.0.0.1
- 不使用 sudo 创建的 pod 是 rootless 的
    - 端口映射默认绑定到 `0.0.0.0`，从主机访问 `localhost:port` 通常可以工作
    - 无法像 rootful 模式那样指定 `127.0.0.1` 或特定网卡 IP 进行绑定
    - 容器之间可以通过 pod 内部的 `localhost` 或容器 IP（一般是 10.88.x.x）互相访问

## 移除失败造成网络问题

有时 podman pod rm 时会出现问题（无法移除某些网络）

此时可能已经造成 podman 网络功能异常（容器无法访问公网，也无法 dns 解析）

解决方法：`sudo reboot`
