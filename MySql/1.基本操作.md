# 通过cli连接到数据库
mysql -h *host* -u *user* -p
- -h 指定数据库所在主机（本地可省略）
- -u 指定用户名
- -p 指定“需要密码”
注意：不能在-p后指定密码  
左侧变为mysql>时，说明成功进入  
输入QUIT可退出cli

# 输入查询
例如：SELECT VERSION(), CURRENT_DATE;
- 语句以分号`;`结尾，才会开始执行
    - 多条语句以分号隔开，可以写在同一行
    - 不写分号就换行，可以写一条多行的语句
    - 写了一半的语句想要取消，写`\c`（在字符串内时无效）
- 大小写不敏感
- prompt状态
    - mysql> 等待语句开始
    - -> 等待语句内的下一行
    - '> "> `> /*> 等待字符串/标识符/注释闭合
## 引号
- 单/双引号表示字符串字面量
- 反引号表示标识符（可解决“标识符是保留字”的问题）

# 创建和重命名数据库
- 创建：`CREATE DATABASE <数据库名>;`
- 重命名：
    注意：以下方法都会丢失表级权限
    - 法1：同设备、同版本（只会移动表结构和数据，丢失外键、触发器、视图等）
        使用`RENAME TABLE`在数据库之间移动表  
        ```sql
        RENAME TABLE db_name.table1 TO new_db_name.table1,
        db_name.table2 TO new_db_name.table2;
        ```
    - 法2：跨设备，跨版本
        使用mysqldump导出数据库，再USE新数据库后使用`SOURCE <sql文件名>`导入
        ```sh
        mysqldump -u xxxx -p db_name > db_name_dump.sql
        ```

# 查看和切换数据库
- `SHOW DATABASES`查看所有数据库
- `USE <数据库名>`进入数据库  
    可在cli连接时，在命令最后指定要直接进入的数据库  
    例如：mysql -h *host* -u *user* -p menagerie
- `SELECT DATABASE()`查看当前在哪个数据库里
- `SHOW TABLES` 查看当前数据库里的所有表

# 创建表
use进入数据库后，可创建表：
    `CREATE TABLE <表名> (<列1名> <列1类型>, <列2名> <列2类型>, ... );`  
    例如：CREATE TABLE pet (name VARCHAR(20), sex CHAR(1), birth DATE);  
`DESCRIBE <表名>`创建表后，可通过DESCRIBE查看其结构

# 修改表
`ALTER TABLE <表名>`
- `ADD <新列名> <数据类型> [<约束>]`  
    ALTER TABLE users ADD email VARCHAR(100) NOT NULL;
- `DROP COLUMN <列名>`  
    ALTER TABLE users DROP COLUMN age;
- `CHANGE <旧列名> <新列名> <新数据类型> <[新约束]>;`  
    ALTER TABLE users CHANGE age user_age INT NOT NULL;
- `MODIFY <列名> <新数据类型> <[新约束]>;`  
    ALTER TABLE users MODIFY age TINYINT UNSIGNED;
- `RENAME TO <新表名>`
    ALTER TABLE users RENAME TO members;

*主键、外键、索引等见“约束”篇*

# 插入数据
- `INSERT INTO <表名> VALUES (<值1>, <值2>, ...);`
    表名后直接跟VALUES，则必须按CREATE TABLE时的顺序，例如：  
    INSERT INTO pet VALUES ('tutuzi', 'Au', 'rabbit', 'm', '2023-8-15', '2025-12-1')
- `INSERT INTO <表名> (<列1名>, <列2名>, ...) VALUES (<值1>, <值2>, ...);`
    可指定字段列表
- 注意：字符串必须有引号，否则会被当做标识符
## 批量插入数据
可使用`LOAD DATA`语句，例：  
LOAD DATA LOCAL INFILE '/path/pet.txt' INTO TABLE pet LINES TERMINATED BY '\r\n';

# 修改数据
`UPDATE <表名> SET <列1名> = <列1新值>, <列2名> = <列2新值>, ... WHERE <条件>;`  
例如：UPDATE pet SET birth = '2025-3-15' WHERE name != 'tutuzi';

# 选择数据
`SELECT * FROM <表名> WHERE <条件>`  
`SELECT [DISTINCT] <表达式1> AS <别名1>, <表达式2>... FROM <表名> WHERE <条件> <排序1>, <排序2>...`  
- SELECT后的星号`*`表示所有列
- SELECT后的DISTINCT：去重
## 条件
- 相等使用单等号`=`
- 条件可用`AND`，`OR`组合，AND的优先级大于OR，但是显式用小括号指定优先级更好
- 字符串比较：一般对大小写不敏感
### NULL
- 筛选NULL，应该使用`IS NULL`和`IS NOT NULL`，不应该使用 = NULL  
    因为任何值（包括NULL）= NULL都返回falsy  
    让NULL与NULL相等的运算符：`<=>`
### 字符串
- 字符串默认大小写不敏感  
    要改为大小写敏感的相等，写：WHERE `BINARY` name = '...'
- 字符串模糊匹配：`LIKE`和`NOT LIKE`（binary有效）
    - `%`表示“多个字符”，例如：“a%”表示a开头的字符串，“%a”表示a结尾的字符串
    - `_`表示“单个字符”，例如：“___”表示长度为3的字符串
- 字符串正则匹配：`REGEXP_LIKE(<列名>, <正则表达式>)`
    - 例如：WHERE REGEXP_LIKE(name, '^m') 表示m开头的字符串
## 排序
`ORDER BY [BINARY] <列名1> [DESC], <列名2>`
- 默认从小到大，列名后面加DESC表示从大到小
- 以字符串列排序时，默认大小写不敏感，BY后加`BINARY`表示大小写敏感
- 多个排序列用逗号分开，DESC仅对其紧跟的列有效
- IS NULL 会被当做 1 和 0
## 别名
当表达式不是列名时，需要重新起名才好读/好用  
使用`<表达式> AS <别名>`，例如：  
SELECT name, TIMESTAMPDIFF(YEAR,birth,CURDATE()) `AS` age FROM pet
## 分组
`GROUP BY <列名1>, <列名2> ...`  
SELECT species, COUNT(*) FROM pets GROUP BY species;  
其中：COUNT充当“聚合函数”，统计每组的某种信息作为一列的输出
## 子查询
`(SELECT ...) AS <别名>` 也可以作为`JOIN`、`SELECT`和`FROM`的目标  
注：子查询必须有AS别名
## 窗口函数
`<函数> OVER (PARTITION BY ...)`  
在不改变原行数的情况下进行分组聚合等计算，每行都得到一个结果，例如：  
SUM(price) OVER (PARTITION BY assets.owner) AS total_price_of_its_owner  
*详情见“窗口函数”篇*
## CTE
`WITH <表达式1> AS <cte名1>, <表达式2> AS <cte名2>...`  
可将select的结果存到CTE（临时结果集），以便后续使用
