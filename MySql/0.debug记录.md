# 查询出的中文是乱码
- 老版本（8之前）的 mysql 的默认字符集是 latin1，无法存储中文。通过以下命令检查：
    ```sh
    mysql -u root -p -e "SHOW VARIABLES LIKE 'character_set_%';"
    mysql -u root -p -e "SHOW VARIABLES LIKE 'collation_%';"
    ```
- 如果发现这个问题，应该修改字符集，然后删空库重新创建，因为已经插入了错误的数据（如果是生产环境，那就坏事了）
- 如果是容器内的mysql，应该修改启动命令：
    ```sh
    docker run -d --name crmeb-mysql \
        -p 3306:3306 \
        -e MYSQL_ROOT_PASSWORD=root \
        -e MYSQL_DATABASE=crmeb \
        mysql:5.7 \
        --character-set-server=utf8mb4 \
        --collation-server=utf8mb4_unicode_ci
    ```

# 导出数据库的大致结构

导出的文件可用于对比（使用 vscode-资源管理器-右键-选择以进行比较）

1. 使用 INTO OUTFILE
    这需要当前用户有 FILE 权限
    ```sql
    SELECT table_name, column_name, data_type FROM information_schema.columns WHERE table_schema = 'kafei_ruohuan_ne' ORDER BY table_name, ordinal_position INTO OUTFILE 'coffee_columns.txt'
    ```
2. 如果没有 FILE 权限，可以直接在终端使用 `mysql -e` 运行表达式，并把结果重定向到文件中
    ```bash
    mysql -u 用户名 -p -e "SELECT table_name, column_name, data_type FROM information_schema.columns WHERE table_schema = '数据库名' ORDER BY table_name, ordinal_position" > 文件名.txt
    ```
3. 如果是 docker 容器内的 mysql，在 `mysql` 前面加 `docker exec 容器名` 即可

# mysql镜像初始化机制

MySQL 官方镜像有一个特殊机制：当数据目录为空时，会自动执行 /docker-entrypoint-initdb.d/ 目录下的 .sql、.sql.gz 或 .sh 脚本。

可以利用这个机制，在 docker-compose.yml 中这样写，让 mysql 实例自动初始化。
```yml
volumes:
  - ./public/install/crmeb.sql:/docker-entrypoint-initdb.d/01-crmeb.sql:ro
```

- 只在首次启动（数据目录为空）时执行
- 执行顺序按文件名排序（01- 前缀确保先执行）
