# network

默认情况下，容器内的 127.0.0.1 指向容器内部，而不是宿主机

- 172.17.0.1 是 Docker/Podman 默认 bridge 网络的网关地址，也就是宿主机的 IP，容器通过这个 IP 访问宿主机
    - 而容器自己被分配到 172.17.0.2， 172.17.0.3 等 IP
    - 在不同版本/环境/网络设置下，这个 IP 可能不同（可移植性差）
- 最好是使用 docker 和 podman 都认识的 `host.containers.internal` 这个特殊域名（推荐）
- 添加 --network host 参数时，容器直接使用宿主机的网络栈，没有独立 IP（无隔离，安全隐患）

# redis

所以，如果有 redis/mysql 实例在监听 127.0.0.1，容器是无法连接的

- 改为监听 0.0.0.0，并设置防火墙阻止外部访问（不安全、麻烦）
- 改为监听容器网关 IP（会变，不推荐）
- 使用 Unix Socket + 卷挂载（推荐）
- 使用容器化 redis 和 mysql（推荐）
    - docker-compose 结合（最省心）
    - 使用同一个 pod，可使用 localhost 互相访问（影响端口映射）
